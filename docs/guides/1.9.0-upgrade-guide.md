# Couchbase Capella Provider 1.9.0: Upgrade and Information Guide

This guide covers what's new, changed, and fixed in version 1.9.0 of the Couchbase Capella Terraform Provider.

## New Features

* Manage cluster deletion protection with [`couchbase-capella_cluster_deletion_protection`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/cluster_deletion_protection)
* The `deletion_protection` attribute is now available on the [`couchbase-capella_cluster`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/cluster) resource and data source, reflecting the cluster's current deletion protection state
* Private endpoints now expose the `private_endpoint_dns` attribute on the [`couchbase-capella_private_endpoints`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/private_endpoints) resource

## Changes

### `tenant_id` is now required for Azure network peers

If you are managing Azure network peers using [`couchbase-capella_network_peer`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/network_peer), the `tenant_id` field inside the `azure_config` block is now required. Previously it was optional. Add this field to your configuration before upgrading if it is missing:

```terraform
resource "couchbase-capella_network_peer" "azure_peer" {
  organization_id = "<organization_id>"
  project_id      = "<project_id>"
  cluster_id      = "<cluster_id>"
  name            = "my-azure-peer"
  provider_type   = "azure"
  provider_config = {
    azure_config = {
      cidr            = "10.6.0.0/16"
      resource_group  = "<resource_group>"
      subscription_id = "<subscription_id>"
      tenant_id       = "<tenant_id>"   # now required
      vnet_id         = "<vnet_id>"
    }
  }
}
```

### `cors.origin` is now required when a CORS block is present on an app endpoint

When configuring CORS on [`couchbase-capella_app_endpoint`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/app_endpoint), the `origin` field inside the `cors` block is now required. If you have a `cors` block without an `origin`, add it before upgrading.

## Bug Fixes

* Fixed a hang where creating a [`couchbase-capella_backup`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/backup) resource could stall for up to 90 minutes due to a race condition in backup polling
* Fixed the `configuration_type` field on clusters being incorrectly handled as a non-pointer, which could cause unexpected plan output
* Fixed cloud snapshot backup pagination, completion detection, and cleanup after restore operations
* Resources are now removed from state when they no longer exist remotely, rather than returning an error
* Rejected private endpoints are no longer retained in state

## New Resources

* [`couchbase-capella_cluster_deletion_protection`](https://registry.terraform.io/providers/couchbasecloud/couchbase-capella/latest/docs/resources/cluster_deletion_protection) - Enable or disable accidental deletion protection on a cluster

## Cluster Deletion Protection

The new `couchbase-capella_cluster_deletion_protection` resource lets you guard a cluster against accidental deletion. When `deletion_protection` is `true`, any attempt to delete the cluster through the Capella API or console will be rejected until protection is turned off.

Destroying this Terraform resource only removes it from state; it does not disable protection on the cluster. To delete a protected cluster, first set `deletion_protection = false` and apply, then destroy.

```terraform
resource "couchbase-capella_cluster_deletion_protection" "example" {
  organization_id     = "<organization_id>"
  project_id          = "<project_id>"
  cluster_id          = "<cluster_id>"
  deletion_protection = true
}
```

For more information, see the [deletion protection examples](https://github.com/couchbasecloud/terraform-provider-couchbase-capella/tree/main/examples/deletion_protection).
